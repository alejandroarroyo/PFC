Maqueta con JADE

  • Inicio/
  • Maquetación

Para maquetar en este entorno se utiliza JADE. Este preprocesador de HTML entre otros aspectos se caracteriza por proporcionar una sintaxis muy sencilla, herencia de plantillas y sentencias de control.

En esta sección se va a hacer una revisión de los mixins proporcionados para mejorar la productividad del desarrollador. Sin embargo el maquetador es libre de crear nuevos mixins acorde con las necesidades de su proyecto.

También se va a describir cómo usar la herencia de plantillas en JADE y se hará un repaso sobre como utilizar las que se proporcionan por defecto en este entorno.

Utiliza la herencia de plantillas

Plantillas

Uno de los puntos fuertes de JADE es la forma que tiene de gestionar cómo se forman las regiones de cada plantilla. A continuación se va a explicar detalladamente cómo utilizar la herencia de plantillas en jade, cómo incluir contenido de otras plantillas y cómo crear regiones con contenido.

Herencia de plantillas y regiones

El fundamento de la herencia de plantilla se basa en los siguientes conceptos:

  1. Crea una plantilla general.
  2. Define regiones en tu plantilla general. Cada región es heredada por otras sub-plantillas.
  3. Cada sub-plantilla rellena las regiones con contenido.

La herencia de plantillas es ilimitada por lo que se puede heredar de tantas plantillas como se desee.

Por lo general los sitios web tienen un par de formas de estructurar el contenido. Por ello la herencia de plantillas es muy útil ya que evita replicar código innecesariamente para estructurar cada vista.

En este entorno se han creado los siguientes layouts para definir la estructura del sitio.

layout.jade: plantilla que define la estructura general de la página. Incluye los mixins, tag html, tag head, el footer y la carga de scripts.

include ../mixins/mixins
include ../includes/head
body
    include ../includes/header
    // CUERPO
section.container
    block pageHeader
    block colLayout
    include ../includes/footer
    include ../includes/scripts

col1layout.jade: esta plantilla hereda el del layout general. Recupera las regiones definidas (blocks) y le agrega el contenido correspondiente para estructurar el contenido en una columna. Además crea una nueva región "content" cuyo contenido será rellenado por la plantilla que herede de esta.

extends layout
block pageHeader
block colLayout
    .row
        .span12
            block content

col2layout.jade: esta plantilla hereda el del layout general. Recuperda las regiones definidas (blocks) y le agrega el contenido correspondiente para estructurar el contenido en dos columnas. Además crea dos nuevas regiones "content" y "sidebar" cuyo contenido será rellenado por la plantilla que herede de esta.

extends layout
block pageHeader
block colLayout
    .row
        .span8
            block content
        .span4
            block sidebar
Definición de términos

A continuación se detallan las diferentes palabras reservadas de los conceptos que se acaban de ver:

  • extends: esta palabra reservada se usa para heredar una plantilla.
  • block: esta palabra reservada se usa para heredar o crear una región.
  • include: esta palabra reservada se usa para importar un .jade en cualquier parte de su plantilla.
¿Cómo creo una plantilla?

Simplemente basta con crear un fichero *.jade en la carpeta "jade" del entorno. A continuación un ejemplo de una plantilla jade a dos columnas.

//- Layout
extends layouts/col1layout
 
//- Encabezado de página
block pageHeader
    h1 Encabezado
    hr
//- Contenido principal
block content
    p Contenido principal
//- Contenido adicional
block sidebar
    p Contenido adicional

Mixins de Jade

Imágenes

mixin img("images/imageExample.png", "Imagen")
mixin img-custom-dimensions("images/imageExample.png", "Imagen", "50px", "75px", "img-circle")

Uso

Por defecto este entorno contiene dos mixins para imágenes.

El siguiente mixin se usa para generar una imagen:
mixin img(imgSrc, imgAlt, imgClass)

  • imgSrc: Recibe la ruta de la imagen.
  • imgAlt: Recibe el texto alternativo de la imagen.
  • imgClass: Atributo opcional para agregar una o más clases a la imagen generada.

El siguiente mixin se usa para generar una imagen con dimensiones personalizdas:
mixin img-custom-dimensions(imgSrc, imgAlt, imgWitdh, imgHeight, imgClass)

  • imgSrc: Recibe la ruta de la imagen.
  • imgAlt: Recibe el texto alternativo de la imagen.
  • imgWitdh: Recibe el ancho de la imagen.
  • imgHeight: Recibe el alto de la imagen.
  • imgClass: Atributo opcional para agregar una o más clases a la imagen generada.

Uso

Por defecto este entorno contiene cinco mixins relacionados con la generación de listas.

El siguiente mixin se usa para generar una lista básica:
mixin list(items, ulClass)

  • items: Contine un array con los elementos de la lista.
  • ulClass: Atributo opcional para agregar una o más clases a la lista generada.

El siguiente mixin se usa para generar una lista básica cuyos elementos son nlaces:
mixin list-links(items, ulClass)

  • items: Contine una colección de arrays con cada elemento. El primer elemento de cada array es el texto que se muestra en la vista y el segundo la URL a la que referencia.
  • ulClass: Atributo opcional para agregar una o más clases a la lista generada.

El siguiente mixin se usa para generar una lista ordenada:
mixin ordered-list(items, olClass)

  • items: Contine un array con los elementos de la lista.
  • olClass: Atributo opcional para agregar una o más clases a la lista generada.

El siguiente mixin se usa para generar una lista básica cuyos elementos son nlaces:
mixin ordered-list-links(items, olClass)

  • items: Contine una colección de arrays con cada elemento. El primer elemento de cada array es el texto que se muestra en la vista y el segundo la URL a la que referencia.
  • olClass: Atributo opcional para agregar una o más clases a la lista generada.

El siguiente mixin se usa para generar una lista de definición:
mixin definition-list(items, dlClass)

  • items: Contine una colección de arrays. Cada sub-array se organiza de la siguiente forma: El primer elemento es el término a definir y los sucesivos son sus definciones.
  • dlClass: Atributo opcional para agregar una o más clases a la lista generada.

Formularios

Input de texto básicos

form(name="formBasicInput")
  mixin input-basic("Input básico", "Introduzca su nombre", "Nombre: ")
  mixin input-basicCustom("Input básico personalizable", "Introduzca sus apellidos", "Apellidos: ", "", "span4")

Uso

Por defecto este entorno contiene dos mixins para crear inputs de texto básicos.

El siguiente mixin se usa para generar un input de texto básico:
mixin input-basic(inputId, placeholder, text)

  • inputId: Atributo "for" del label e "id" del input.
  • placeholder: "placeholder" del input.
  • text: Texto del label.

El siguiente mixin se usa para generar una input de texto básico personalizable:
mixin input-basicCustom(inputId, placeholder, text, customLabelClass, customInputClass)

  • inputId: Atributo "for" del label e "id" del input.
  • placeholder: "placeholder" del input.
  • text: Texto del label.
  • customLabelClass: Atributo opcional para agregar una o más clases al label.
  • customInputClass: Atributo opcional para agregar una o más clases al input.

Componentes avanzados

form(name="formAdvanced")
    mixin input-prepended("$", "minMoney")
    mixin input-appended("$", "maxMoney")
    mixin input-combined("$", "finalMoney", ",000")
    mixin input-button-appended("Iniciar sesión", "loggin")
    mixin input-button-prepended("Cerrar sesión", "disconnect")
form(name="formAdvanced2", class="form-search")
    mixin input-prepended-rounded("Enviar", "send")
    mixin input-appended-rounded("Aceptar", "accept", "", "btn-inverse")

Uso

Por defecto este entorno contiene varios mixins para crear inputs especiales.

El siguiente mixin se usa para generar un input con addon al principio del campo:
mixin input-prepended(addon, inputName, wrapperClass)

  • addon: Recibe el addon que se inserta al inicio del input.
  • inputName: Atributo "name" del input.
  • wrapperClass: Parámetro opcional para agregar una clase al contenedor del input.

El siguiente mixin se usa para generar un input con addon al final del campo:
mixin input-appended(addon, inputName, wrapperClass)

  • addon: Recibe el addon que se inserta al final del input.
  • inputName: Atributo "name" del input.
  • wrapperClass: Parámetro opcional para agregar una clase al contenedor del input.

El siguiente mixin se usa para generar un input con addon al principio y al final del campo:
mixin input-combined(firstAddon, inputName, secondAddon, wrapperClass)

  • firstAddon: Recibe el addon que se inserta al principio del input.
  • inputName: Atributo "name" del input.
  • secondAddon: Recibe el addon que se inserta al final del input.
  • wrapperClass: Parámetro opcional para agregar una clase al contenedor del input.

El siguiente mixin se usa para generar un input con un botón al principio del campo:
mixin input-button-prepended(text, inputName, wrapperClass)

  • text: Recibe el texto del botón.
  • inputName: Atributo "name" del input.
  • wrapperClass: Parámetro opcional para agregar una clase al contenedor del input y el botón.

El siguiente mixin se usa para generar un input con un botón al final del campo:
mixin input-button-appended(text, inputName, wrapperClass)

  • text: Recibe el texto del botón.
  • inputName: Atributo "name" del input.
  • wrapperClass: Parámetro opcional para agregar una clase al contenedor del input y el botón.

El siguiente mixin se usa para generar un input redondeado con un botón al principio del campo:
mixin input-prepended-rounded(buttonText, inputName, customInputClass, customButtonClass)

  • buttonText: Recibe el texto del botón.
  • inputName: Atributo "name" del input.
  • customInputClass: Parámetro opcional para agregar una clase al input.
  • customButtonClass: Parámetro opcional para agregar una clase al botón.

El siguiente mixin se usa para generar un input redondeado con un botón al final del campo:
mixin input-appended-rounded(buttonText, inputName, customInputClass, customButtonClass)

  • buttonText: Recibe el texto del botón.
  • inputName: Atributo "name" del input.
  • customInputClass: Parámetro opcional para agregar una clase al input.
  • customButtonClass: Parámetro opcional para agregar una clase al botón.
