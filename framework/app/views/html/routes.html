En este entorno se utiliza AngularJS. Entre muchas de las ventajas que ofrece este framework MVC de javascript se encuentra la capacidad de crear aplicaciones web con una rica interacción y favorecer las URLs amigables.

En esta sección se va a describir como funciona el servicio $routeProvider de AngularJS que permite mostrar un contenido u otro y asignar lógica a esa parte de la vista en caso de ser necesario.

Servicio $routeProvider

index.jade

Con el entorno se proporciona un fichero index.jade (el cual genera su correspondiente .html) con el siguiente contenido.

!!! 5
html(lang="es", class="no-js", ng-app="projectApp")
    include includes/head
    body(style="padding: 0;")
        .container(ng-view)
    include includes/scripts

Este index es el tronco de la aplicación web y donde se va a cargar contenido. De este fichero se puede destacar lo siguiente:

ng-app
Módulo de la aplicación. Esta directiva quiere decir que se va a usar AngularJS dentro del tag html.
ng-view
Directiva de Angular que se encarga de cargar un contenido u otro. Recoge información del servicio $routeProvider del fichero app.js que se proporciona. Cuando AngularJS detecta que una ruta cambia remplaza esta directiva por el contenido correspondiente.

app.js

Ahora vamos a echar un vistazo al fichero app.js y su servicio de enrutado.

'use strict';
angular.module('projectApp', [])
    .config(function ($routeProvider) {
        $routeProvider
            .when('/', {
                templateUrl: 'views/html/home.html'
            })
            .when('/welcome', {
                templateUrl: 'views/html/welcome.html'
            })
            .when('/main', {
                templateUrl: 'views/html/main.html',
                controller: 'MainCtrl'
            })
            .when('/first-steps', {
                templateUrl: 'views/html/first-steps.html'
            })
            .when('/design', {
                templateUrl: 'views/html/design.html'
            })
            .when('/routes', {
                templateUrl: 'views/html/routes.html'
            })
            .when('/accessibility', {
                templateUrl: 'views/html/accessibility.html'
            })
            .otherwise({
                redirectTo: '/'
            });
    });

Con servicio$routeProviderdada una ruta relativa al sitio web hará diferentes cosas. En cada método whense especifica una ruta relativa y se puede personaliza con diferentes opciones. Por otro lado en el método otherwisese especifica qué hacer en caso de no haber cargado ninguna de las rutas anteriores. Este servicio favorece las rutas amigables ya que la dirección mostrada en navegador es la que está dentro del when.

templateUrl
Especifica que la ruta de la plantilla a cargar.
controller
Especifica el controlador para esa vista. De este modo es posible agregar lógica a una plantilla.
redirectTo
Se proporciona una ruta a la que navegar.
resolve
Este objeto le dice avisa a AngularJS de que algo debe ser cargado antes de mostrar la plantilla al usuario. Es útil en caso de realizar peticiones al servidor.
